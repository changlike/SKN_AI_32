 **FastAPI + ReAct + RAG + A2A + LangGraph + MCP 통합 프로젝트**

## 적용 구조

```text
사용자
  ↓
FastAPI 웹 UI / REST API / Swagger
  ↓
LangGraph StateGraph
  ├─ ReAct
  │    └─ LLM이 도구 선택 → 실행 → 결과 관찰 → 최종 답변
  │
  ├─ RAG
  │    └─ 정책 PDF → 청크 분할 → 임베딩 → FAISS → 근거 답변
  │
  ├─ A2A
  │    ├─ order-agent
  │    ├─ inventory-agent
  │    ├─ faq-agent
  │    ├─ policy-agent
  │    └─ exchange-agent
  │
  └─ MCP
       ├─ 주문 조회 도구
       ├─ 재고 조회 도구
       ├─ FAQ 검색 도구
       └─ 교환 신청 도구
  ↓
OpenAI 또는 Gemini
  ↓
data.zip 실데이터 / 정책 PDF
```

자료의 주문·재고·FAQ 실데이터 도구, 정책 RAG, 단기 메모리, 멀티턴 시나리오, 오류 처리 및 교환신청 실습을 FastAPI 구조로 확장.

## 1. 공통 모듈 `common.py`

기존에 제공된 `common.py`를 FastAPI 프로젝트 구조에 맞게 경로만 조정해 사용.

```python
from app.core.common import (
    ROOT,
    DATA,
    DOCS,
    get_chat,
    get_embeddings,
    get_genai_client,
)
```

공통 처리 항목:

```text
.env 로드
OpenAI API 키 검사
Gemini API 키 검사
OpenAI 채팅 모델 생성
Gemini 채팅 모델 생성
OpenAI 임베딩 생성
Gemini 임베딩 생성
data 및 docs 경로 제공
```

`llm_factory.py`는 모델을 별도로 중복 생성하지 않고 반드시 `common.py`의 `get_chat()`과 `get_embeddings()`를 호출합니다.

## 2. ReAct 구조

`app/agents/react_agent.py`에 구현했습니다.

```text
사용자 질문
  ↓
질문 분석
  ↓
도구 선택
  ↓
도구 실행
  ↓
실행 결과 관찰
  ↓
추가 도구 실행 또는 최종 답변
```

사용 가능한 도구:

```text
get_order_status
get_stock
search_faq
policy_search
request_exchange
```

최신 LangChain에서는 기존 `langgraph.prebuilt.create_react_agent`보다 `langchain.agents.create_agent` 사용이 권장되므로 해당 방식으로 구현했습니다. LangGraph는 그보다 낮은 수준에서 상태, 분기, 영속 실행을 제어하도록 역할을 분리했습니다. ([Docs by LangChain][1])

## 3. RAG 구조

`app/services/rag_service.py`에 구현했습니다.

```text
data/docs/*.pdf
  ↓
PyPDFLoader
  ↓
RecursiveCharacterTextSplitter
  ↓
OpenAI 또는 Gemini Embeddings
  ↓
FAISS Vector Store
  ↓
similarity_search
  ↓
출처와 페이지가 포함된 검색 근거
  ↓
근거 고정 최종 답변
```

PDF 이름을 직접 지정하지 않고 다음처럼 전체 PDF를 자동 검색합니다.

```python
pdf_files = sorted(DOCS_DIR.glob("*.pdf"))
```

따라서 압축 해제 과정에서 정책 PDF 이름이 인코딩된 형태로 표시되더라도 정상 처리할 수 있습니다.

정책 질문은 다음 원칙으로 동작합니다.

```text
정책 문서 근거가 있으면 근거 기반 답변
정책 문서에 없으면 확인할 수 없다고 답변
없는 정책을 임의로 생성하지 않음
```

## 4. A2A 구조

`app/agents/specialists.py`에 전문 에이전트를 구현했습니다.

```text
order-agent
inventory-agent
faq-agent
policy-agent
exchange-agent
```

각 에이전트는 에이전트 카드를 가집니다.

```json
{
  "name": "policy-agent",
  "description": "정책 RAG 전문 에이전트",
  "skills": [
    "환불 정책",
    "교환 정책",
    "멤버십 정책"
  ],
  "endpoint": "/api/v1/a2a/message",
  "version": "1.0.0"
}
```

A2A는 서로 다른 에이전트 애플리케이션이 에이전트 카드로 기능을 공개하고 메시지 및 작업을 교환하는 공개 표준입니다. 프로젝트에는 에이전트 카드 조회와 역할별 메시지 위임 API를 적용했습니다. ([GitHub][2])

## 5. LangGraph 구조

`app/graph/workflow.py`에 구현했습니다.

```text
START
  ↓
classify
  ├── 정책 질문
  │     ↓
  │    RAG
  │     ↓
  │    grounded_answer
  │     ↓
  │    END
  │
  ├── 교환·재고 질문
  │     ↓
  │    A2A
  │     ↓
  │    END
  │
  ├── 주문번호 질문
  │     ↓
  │    MCP
  │     ↓
  │    END
  │
  └── 복합 질문
        ↓
       ReAct
        ↓
       END
```

워크플로우 상태:

```python
class WorkflowState(TypedDict, total=False):
    message: str
    thread_id: str
    provider: Literal["openai", "gemini"]
    route: str
    target_agent: str
    context: str
    answer: str
    error: str
    trace: list[dict[str, object]]
```

각 노드는 `trace`에 실행 기록을 남깁니다.

```json
[
  {
    "stage": "Start",
    "detail": "요청 시작: attempt=1"
  },
  {
    "stage": "LangGraph",
    "detail": "주문번호 질문을 MCP 경로로 분류"
  },
  {
    "stage": "MCP",
    "detail": "get_order_status 도구 호출 완료"
  },
  {
    "stage": "Complete",
    "detail": "총 처리시간=0.023초"
  }
]
```

## 6. MCP 구조

독립 MCP 서버:

```text
app/mcp_server/server.py
```

MCP 도구:

```text
mcp_order_status
mcp_stock
mcp_faq
mcp_request_exchange
```

실행 명령:

```bash
python -m app.mcp_server.server
```

기본 전송 방식은 `stdio`입니다.

```python
mcp.run(transport="stdio")
```

MCP는 모델이 외부 데이터와 기능을 표준화된 도구 형태로 발견하고 호출하게 하는 프로토콜이며, 공식 Python SDK는 서버·클라이언트·도구·리소스 및 표준 전송 방식을 지원합니다. ([GitHub][3])

## 7. FastAPI 엔드포인트

| 메서드  | URL                          | 기능              |
| ---- | ---------------------------- | --------------- |
| GET  | `/`                          | 웹 채팅 UI         |
| GET  | `/docs`                      | Swagger         |
| GET  | `/redoc`                     | ReDoc           |
| GET  | `/api/v1/health`             | 서버 및 API 키 상태   |
| POST | `/api/v1/chat`               | 전체 통합 워크플로우     |
| POST | `/api/v1/tools/call`         | 개별 MCP 호환 도구 실행 |
| GET  | `/api/v1/a2a/agents`         | A2A 에이전트 카드     |
| POST | `/api/v1/a2a/message`        | 전문 에이전트 위임      |
| GET  | `/api/v1/data/files`         | data.zip 데이터 목록 |
| POST | `/api/v1/rag/reset`          | FAISS 캐시 초기화    |
| POST | `/api/v1/exercises/exchange` | 교환신청 실습 해답      |
| GET  | `/api/v1/exercises/memory`   | 메모리 격리 실습       |

FastAPI 프로젝트는 라우터 기반 다중 파일 구조와 lifespan 초기화 방식으로 작성했습니다. ([FastAPI][4])

## 9. PyCharm 실행 방법

### 가상환경 생성

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

패키지 설치:

```bash
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 환경변수 설정

`.env.example`을 `.env`로 복사합니다.

```env
OPENAI_API_KEY=본인의_OpenAI_API_KEY
OPENAI_MODEL=gpt-4o-mini
OPENAI_EMBED_MODEL=text-embedding-3-small

GOOGLE_API_KEY=본인의_Google_API_KEY
GEMINI_MODEL=gemini-2.5-flash
GEMINI_EMBED_MODEL=models/gemini-embedding-001
```

### 실행

PyCharm에서 `run.py`를 우클릭하여 실행합니다.

터미널:

```bash
python run.py
```

접속:

```text
웹 UI   : http://127.0.0.1:8000
Swagger : http://127.0.0.1:8000/docs
ReDoc   : http://127.0.0.1:8000/redoc
```
