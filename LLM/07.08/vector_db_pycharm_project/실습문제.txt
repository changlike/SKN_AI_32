실습문제
직접 코드를 고쳐 보며 내용을 손에 익힙니다.  (이 실습은 GOOGLE_API_KEY가 필요합니다.)

문제 1 — FAISS 인덱스 생성·저장
data/docs/멤버십정책.pdf를 청킹·임베딩하여 FAISS 인덱스를 만들고 
data/faiss_index/membership에 save_local하세요.

기대 결과: 저장 폴더에 index.faiss, index.pkl이 생성되고, 저장된 청크 개수를 출력하면 성공.


문제 2 — load 후 검색 검증
문제 1에서 저장한 인덱스를 load_local로 불러와 "VIP 적립률"을 검색하세요.

사용 파일: data/faiss_index/membership
기대 결과: 적립률 정보가 든 청크가 top-1으로 검색되고, 
새로 만들지 않고 로드만으로 검색됨을 확인하면 성공.


문제 3 — metadata 출처 추적
검색 답변에 문서파일명과 페이지 정보도 함께 출력되게 하세요.


문제 4 — 인덱스 증분 업데이트
새 문서로 '환불교환정책.pdf'를 추가하는 증분 업데이트를 구현하세요.
검색 질문으로 추가된 문서의 내용도 검색 결과에 포함되는지 출력 되게 하세요.


문제 5 — 메타데이터 필터링
메뉴 실행시 두 개의 pdf 파일명을 제공하고 선택하여, 선택한 파일에서만 질문이 검색되게 필터링하세요.

1. 멤버십정책.pdf
2. 환불교환정책.pdf

선택한 문서파일명과 페이지
검색 결과 출력되게 하세요.


정리
문제 1로 인덱싱 + 저장(build_index_genai.py)을 만듭니다.
문제 2로 로드 + 검색(service_genai.py)을 만들어 둘을 분리합니다.
문제 3은 문제 2에 내용 복사한 다음, 메타데이터 출력(print_metadata_genai.py)을 추가합니다.
문제 4에 인덱스 로드해서 인덱스 증분 업데이트(incremental_update_genai.py)합니다.
문제 5로 문제4의 저장 결과 로드 + 메타데이터 필터링 검색(filter_search_genai.py) 구현합니다.
