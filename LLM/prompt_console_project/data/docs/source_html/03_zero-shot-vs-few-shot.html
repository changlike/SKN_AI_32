zero-shot vs few-shot

한 줄 요약

zero-shot은 예시 없이 지시만 주는 것, few-shot은 정답 예시 몇 개를 먼저 보여 주고 시키는 것입니다. few-shot은 경계가 헷갈리는 분류·판단에서 정확도를 크게 올려 줍니다.

1. 두 방식의 차이

zero-shot : (예시 없이) 지시만 주고 바로 시킨다
            "다음 문의를 분류하라: '카드가 두 번 청구됐어요'"

few-shot  : 정답 예시 몇 개를 먼저 보여주고 시킨다
            예) '반품 배송비 누가 내요?'      → 환불
                '색상이 사진과 달라요, 교환?'  → 교환
            그 다음에  "그럼 이건? '카드가 두 번 청구됐어요'"

few-shot은 모델에게 "이런 식으로 답해" 라는 본보기를 주는 것입니다.

비유: 신입에게 "분류해"라고만 하면(zero-shot) 자기 기준대로 합니다. 하지만 "이건 이렇게, 저건 저렇게 분류한 거야. 같은 식으로 해"라며 예시 몇 개를 보여 주면(few-shot), 우리 회사 기준을 금방 따라옵니다.

2. few-shot 프롬프트 만들기

분류 작업에 few-shot을 적용해 봅니다. 예시 4개를 프롬프트 안에 넣습니다.

CATEGORIES = ["배송", "환불", "교환", "결제", "상품문의", "칭찬", "불만"]

# few-shot = 정답 예시 몇 개를 먼저 보여주고 같은 식으로 답하게 하는 기법.
FEWSHOT = """다음 고객 문의를 아래 7개 중 정확히 하나로 분류하라.
카테고리: 배송 / 환불 / 교환 / 결제 / 상품문의 / 칭찬 / 불만
카테고리 이름 한 단어만 출력하라(다른 말 금지).

[예시]
문의: 반품하면 배송비는 누가 부담하나요?           → 환불
문의: 색상이 사진과 달라요. 다른 색으로 바꿔주세요.  → 교환
문의: 상담원분이 정말 친절하셨어요. 감사합니다.      → 칭찬
문의: 카드가 두 번 청구됐어요.                      → 결제
"""

이 프롬프트에는 4요소가 다 들어 있습니다 — 지시("분류하라"), 맥락("카테고리 7개"), 형식("한 단어만"), 그리고 예시 4개(few-shot).

3. 분류 함수

from common import get_genai_client, GEMINI_MODEL
from google.genai import types

client = get_genai_client()

def classify(content: str) -> str:
    """문의 한 건을 7개 카테고리 중 하나로 분류한다(few-shot 사용)."""
    resp = client.models.generate_content(
        model=GEMINI_MODEL,
        contents=f"{FEWSHOT}\n[분류할 문의]\n문의: {content} →",
        config=types.GenerateContentConfig(temperature=0),  # 분류는 일관성이 중요 → 0
    )
    out = resp.text.strip()
    for c in CATEGORIES:        # 군더더기가 붙어도 7개 중 포함된 단어를 골라낸다
        if c in out:
            return c
    return "기타"

두 가지 포인트가 있습니다.

  • temperature=0: 분류는 매번 같은 답이 나와야 하므로 무작위성을 끕니다(2강 복습).
  • for c in CATEGORIES: 모델이 "결제입니다" 처럼 군더더기를 붙여도, 7개 중 포함된 단어를 골라내는 안전망입니다.

4. 실행

print(classify("카드가 두 번 청구됐어요"))        # → 결제
print(classify("포장이 찢어진 채로 왔어요"))       # → 불만 또는 교환
print(classify("이 제품 방수 되나요?"))            # → 상품문의

예상 출력

결제
불만
상품문의

예시를 보여 줬더니, 우리가 원하는 7개 카테고리 안에서 한 단어로 깔끔하게 답합니다.

5. few-shot, 예시는 몇 개가 좋을까?

  • 너무 적으면(0~1개): 모델이 기준을 못 잡습니다.
  • 적당하면(3~5개): 대부분의 분류 작업에 충분합니다.
  • 너무 많으면: 토큰(비용)이 늘고, 효과는 점점 줄어듭니다.

특히 헷갈리는 경계(환불 vs 결제, 불만 vs 상품문의)를 콕 집은 예시가 효과적입니다. 이 "경계 예시 추가"는 07번(오분류 분석)과 연습문제에서 직접 해 봅니다.

핵심 정리

  • zero-shot = 예시 없음, few-shot = 정답 예시를 보여 줌.
  • few-shot은 경계가 헷갈리는 분류에서 정확도를 크게 올린다.
  • 분류는 temperature=0으로 일관성을 확보한다.
  • 예시는 3~5개, 특히 헷갈리는 경계를 집은 예시가 효과적이다.
