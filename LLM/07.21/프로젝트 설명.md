[Business Agent FastAPI 최종 프로젝트 다운로드]

## 적용 구조

```text
사용자
  ↓
FastAPI 웹 UI / REST API / Swagger
  ↓
LangGraph StateGraph
  ├─ ReAct
  │    └─ 질문 분석 → 도구 선택 → 실행 → 관찰 → 답변
  │
  ├─ RAG
  │    └─ PDF·CSV → 청크 분할 → 임베딩 → FAISS → 근거 검색
  │
  ├─ A2A
  │    ├─ sales-analyst-agent
  │    ├─ report-writer-agent
  │    ├─ knowledge-agent
  │    └─ strategy-agent
  │
  └─ MCP
       ├─ 월별 매출 조회
       ├─ CSV 미리보기
       ├─ 데이터 구성 요약
       └─ 전체 데이터 파일 조회
  ↓
OpenAI 또는 Gemini
  ↓
monthly_sales.csv / 정책 PDF / 제품 매뉴얼 / 기타 비즈니스 데이터
```

## 프로젝트 기능

* 최신 월 매출 집계
* 특정 월과 전월 비교
* 전월 대비 성장률 계산
* 카테고리별 매출 순위
* 파일·컬럼·누락값 예외 처리
* OpenAI 또는 Gemini 보고서 작성
* LLM 장애 시 기본 보고서 폴백
* 마크다운 보고서 저장
* 카테고리별 매출 PNG 차트 생성
* 차트를 포함한 보고서 생성
* 원본 PDF·CSV·JSON 데이터 유지

## LangGraph 자동 분기

질문의 내용에 따라 실행 경로가 자동 선택됩니다.

```text
정책 / 매뉴얼 / 핸드북 / 내부 근거
→ RAG
→ knowledge-agent

보고서 / 리포트 / 차트 / 그래프
→ A2A
→ report-writer-agent

매출 / 성장률 / 카테고리 / 전월 비교
→ A2A
→ sales-analyst-agent

전략 / 마케팅 / 개선 / 제언
→ A2A
→ strategy-agent

mcp: 접두어
→ MCP 도구 직접 실행

그 밖의 복합 질문
→ ReAct
→ 필요한 도구 자율 선택
```

## ReAct 도구

ReAct 에이전트는 다음 도구를 질문에 따라 선택합니다.

```text

internal_knowledge_search
```

예를 들어 다음 질문은 여러 도구를 연속으로 사용할 수 있습니다.

```text

```

## RAG 구조

```text
data/docs/*.pdf
핵심 CSV
  ↓
Document 변환
  ↓
RecursiveCharacterTextSplitter
  ↓
OpenAI 또는 Gemini Embeddings
  ↓
FAISS
  ↓
유사 문서 검색
  ↓
출처가 포함된 근거 고정 답변
```

RAG에 포함되는 주요 자료:

* 환불교환정책
* 멤버십정책
* 직원핸드북
* 제품 매뉴얼
* 월별 매출
* 경쟁사 데이터
* 마케팅 브리프
* 상품 데이터
* 리뷰 데이터

## A2A 전문 에이전트

### `sales-analyst-agent`

* 최신 월 매출
* 특정 월 매출
* 전월 대비 증감률
* 카테고리 순위
* 확정 수치 출력

### `report-writer-agent`

* 경영진용 보고서 작성
* 마크다운 파일 저장
* 카테고리 차트 생성
* 차트 포함 보고서 생성

### `knowledge-agent`

* 정책 PDF 검색
* 제품 매뉴얼 검색
* CSV 근거 검색
* 출처 기반 답변

### `strategy-agent`

* 매출 facts와 내부 문서 결합
* 마케팅 및 운영 제언
* 근거 없는 숫자 생성 방지
* 실행 가능한 경영 전략 작성

## 주요 FastAPI API

| 메서드  | URL                                   | 기능                  |
| ---- | ------------------------------------- | ------------------- |
| GET  | `/`                                   | Business Agent 웹 UI |
| GET  | `/docs`                               | Swagger             |
| GET  | `/redoc`                              | ReDoc               |
| GET  | `/api/v1/health`                      | 서버·API 키·구조 상태      |
| POST | `/api/v1/chat`                        | LangGraph 통합 실행     |
| POST | `/api/v1/tools/call`                  | MCP 호환 도구 호출        |
| GET  | `/api/v1/a2a/agents`                  | A2A 전문 에이전트 카드      |
| GET  | `/api/v1/.well-known/agent-card.json` | 통합 Agent Card       |
| POST | `/api/v1/a2a/message`                 | 전문 에이전트 직접 위임       |
| GET  | `/api/v1/data/files`                  | 전체 데이터 파일 목록        |
| POST | `/api/v1/rag/reset`                   | FAISS 인덱스 초기화       |
| GET  | `/api/v1/reports/{filename}`          | 보고서·차트 다운로드         |

## PyCharm 실행 방법

### 1. 가상환경 생성

```powershell
python -m venv .venv
```

### 2. 가상환경 활성화

```powershell
.\.venv\Scripts\Activate.ps1
```

### 3. 패키지 설치

```powershell
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### 4. 환경변수 파일 생성

```powershell
Copy-Item .env.example .env
```

`.env` 파일에 API 키를 입력합니다.

```dotenv
OPENAI_API_KEY=실제_OpenAI_API_KEY
OPENAI_MODEL=gpt-4o-mini
OPENAI_EMBED_MODEL=text-embedding-3-small

GOOGLE_API_KEY=실제_Google_API_KEY
GEMINI_MODEL=gemini-2.5-flash
GEMINI_EMBED_MODEL=models/gemini-embedding-001
```

### 5. 실행

```powershell
python run.py
```

접속 주소:

```text
웹 UI   : http://127.0.0.1:8000
Swagger : http://127.0.0.1:8000/docs
ReDoc   : http://127.0.0.1:8000/redoc
```
